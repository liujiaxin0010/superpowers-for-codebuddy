# 编写计划（Writing Plans）

将已确认设计转换为可执行任务计划。写计划前必须先通过门禁检查。

## 硬门禁前置条件

先运行 `.codebuddy/skills/process-gatekeeper/SKILL.md`。

若 `GateResult.status=blocked`：

- 输出阻断结果
- 立即停止

## 输出路径

`docs/plans/YYYY-MM-DD-<feature>.md`

## 计划中必填元数据

```yaml
specPath: docs/specs/...
finalTier: L|M|H
gateStatus: pass
```

## 要求

1. 任务粒度：每项 2-5 分钟
2. 每个任务必须包含测试与验证命令
3. 明确依赖顺序与可并行任务
4. 必须包含风险缓解章节
5. 计划前必须确认 spec 的“需求澄清结论”与“方案方向确认”已完成，且不存在 `TBD/待定/未确认`
6. 若用户否决候选方向，必须记录明确替代方向与硬约束，否则不得拆解计划
7. 计划任务中必须包含日志相关改动与验证：
   - 旧项目：复用原有日志结构
   - 新项目：落实已选日志框架与字段规范
   - 验证无中文日志、无控制台输出残留（除非用户要求）

## 分级附加要求

- L：最小可执行任务链
- M：补充明确风险缓解任务，并强制代码审查阶段
- H：必须具备头脑风暴证据，并包含回滚演练任务
