# 审查检查项清单

按 5 大类别组织的详细检查项。审查时逐类扫描，确保不遗漏。

---

## 类别1: 代码逻辑类

### 1.1 函数与方法

- [ ] 每个函数是否只做一件事（单一职责）？超过 50 行的函数是否应拆分？
- [ ] 函数参数是否有校验？特别是来自用户输入 / API 响应 / 路由参数的数据
- [ ] 函数返回值是否被正确处理？Promise 是否有 `.catch()` 或 `try-catch`？
- [ ] async 函数是否都有对应的 `await`？是否有未等待的 Promise？
- [ ] 回调函数中 `this` 指向是否正确？Vue 2 中是否使用了箭头函数导致 this 丢失？

### 1.2 条件与分支

- [ ] if/else 条件逻辑是否正确？是否有取反错误？
- [ ] switch 语句是否有 default 分支？是否有遗漏的 break？
- [ ] 边界条件是否处理？（null / undefined / 空数组 / 空字符串 / 0 / false）
- [ ] 三元表达式是否清晰？嵌套三元是否应改为 if/else？
- [ ] 可选链 `?.` 是否使用得当？是否有遗漏的空值判断？

### 1.3 数据与状态

- [ ] Vue data 中的响应式数据是否正确声明？是否有直接赋值新属性（应使用 `$set`）？
- [ ] Vuex state 的修改是否通过 mutation？是否有直接修改 state 的情况？
- [ ] 深拷贝是否正确？是否有引用类型的浅拷贝导致数据污染？
- [ ] 数组操作是否使用 Vue 能检测到的方法？（push/splice/Vue.set 而非下标直接赋值）

### 1.4 安全性

- [ ] **v-html** 是否使用了未过滤的用户输入？（XSS 风险 → Major）
- [ ] 是否有 `eval()` / `new Function()` / `innerHTML` 使用？
- [ ] 是否有硬编码的 API Key / Token / 密码？
- [ ] 是否有 `console.log` 输出敏感信息（在生产环境中）？
- [ ] 第三方库的输入是否做了安全过滤？

### 1.5 类型安全（TS 文件）

- [ ] 是否滥用 `any` 类型？
- [ ] 接口/类型定义是否完整？是否有遗漏的可选属性？
- [ ] 泛型使用是否正确？
- [ ] 类型断言（as）是否安全？

---

## 类别2: 视觉呈现类缺陷

### 2.1 CSS / 样式

- [ ] 是否使用了 `scoped` 避免样式污染？（Vue 组件）
- [ ] 是否有 `!important` 滥用？
- [ ] 是否有冗余或重复的 CSS 属性？
- [ ] z-index 是否合理？是否有"z-index 大战"？
- [ ] 颜色、字体、间距是否使用设计系统的变量/token？

### 2.2 布局

- [ ] flex/grid 布局是否正确？是否有溢出问题？
- [ ] 固定宽度/高度是否会在不同屏幕下出问题？
- [ ] 文本过长时是否有 text-overflow / word-break 处理？
- [ ] 图片是否设置了宽高或 aspect-ratio 防止布局抖动？

### 2.3 响应式

- [ ] 是否考虑了移动端适配？
- [ ] 媒体查询断点是否与设计规范一致？
- [ ] 触摸设备上的交互是否正常？（hover 状态在移动端不可用）

### 2.4 动画

- [ ] 动画是否使用 `transform` / `opacity` 而非 `top` / `left` / `width`？
- [ ] 是否有 `will-change` 优化？
- [ ] transition/animation 在组件销毁时是否正确清理？

---

## 类别3: 交互体验类缺陷

### 3.1 事件处理

- [ ] 按钮点击事件是否绑定正确？
- [ ] 是否有事件冒泡/捕获问题？需要 `.stop` / `.prevent` 的场景？
- [ ] 高频事件（input/scroll/resize）是否做了**防抖/节流**？
- [ ] 键盘事件是否处理？（回车提交、Esc 关闭等）

### 3.2 用户反馈

- [ ] 异步操作（API 请求）是否有 **loading 状态**？
- [ ] 操作成功/失败是否有**明确提示**？（$message / Toast）
- [ ] 错误信息是否对用户友好？（不暴露技术细节）
- [ ] 空状态（空列表/无数据）是否有**占位提示**？

### 3.3 表单处理

- [ ] 必填项是否有**验证规则**？
- [ ] 输入格式是否有校验？（邮箱、手机号、身份证等）
- [ ] 提交按钮是否在**请求中禁用**防止重复提交？
- [ ] 表单重置是否清除所有状态（包括验证状态）？
- [ ] 长表单是否有分步引导？

### 3.4 导航与路由

- [ ] 路由守卫中的权限检查是否正确？
- [ ] 路由参数变化时组件是否正确响应？（watch $route）
- [ ] 返回按钮的行为是否符合预期？
- [ ] 404 / 无权限页面是否正确处理？

### 3.5 无障碍（A11y）

- [ ] 图片是否有 alt 属性？
- [ ] 表单元素是否有 label 关联？
- [ ] 颜色对比度是否达标？
- [ ] 自定义组件是否有 ARIA 属性？

---

## 类别4: 性能表现类缺陷

### 4.1 内存管理

- [ ] `addEventListener` 是否在 `beforeDestroy` / `onUnmounted` 中移除？
- [ ] `setInterval` / `setTimeout` 是否在组件销毁时清除？
- [ ] EventBus `$on` 是否有对应的 `$off`？
- [ ] 大对象引用是否在不需要时置空？
- [ ] 闭包是否持有了不必要的外部引用？

### 4.2 渲染性能

- [ ] `v-if` 和 `v-for` 是否**没有同时使用**在同一元素上？
- [ ] `v-for` 是否有 `:key` 且 key 是唯一标识（非 index）？
- [ ] computed 属性是否只依赖了必要的响应式数据？
- [ ] 大列表（>100 条）是否使用了**虚拟滚动**？
- [ ] 组件是否有不必要的重渲染？（可用 `v-once` / `Object.freeze` 优化）
- [ ] 复杂计算是否放在 computed 而非 template 中？

### 4.3 网络性能

- [ ] 是否有重复的 API 请求？（特别是在 watch / computed 中）
- [ ] 大文件上传是否有分片处理？
- [ ] 图片是否做了懒加载？
- [ ] 路由组件是否使用了按需加载（`() => import()`）？
- [ ] API 响应是否有必要的缓存？

### 4.4 算法效率

- [ ] 是否有 O(n²) 及以上复杂度可优化的循环？
- [ ] 数组查找是否可以用 Map/Set 替代线性查找？
- [ ] 正则表达式是否有性能问题（回溯爆炸）？

---

## 类别5: 内容准确性类缺陷

### 5.1 文案与拼写

- [ ] 是否有中文错别字？
- [ ] 是否有英文拼写错误？（变量名、注释、提示信息）
- [ ] 提示信息是否准确描述了操作结果？
- [ ] 标点符号是否正确？（中文用中文标点，英文用英文标点）

### 5.2 国际化（i18n）

- [ ] 界面文字是否都使用了 `$t()` 或 i18n 函数？
- [ ] 是否有硬编码的中文字符串？
- [ ] 日期/时间/数字格式是否支持国际化？
- [ ] 错误提示是否使用了 i18n key？

### 5.3 数据展示

- [ ] 数字是否正确格式化？（千分位、小数位、百分比）
- [ ] 日期格式是否统一？（YYYY-MM-DD vs DD/MM/YYYY）
- [ ] 枚举值是否正确映射为中文？
- [ ] 空值是否有默认展示？（"—" 或 "暂无数据"）
- [ ] 单位是否正确？（元 vs 万元、KB vs MB）

### 5.4 注释与文档

- [ ] 复杂逻辑是否有注释说明？
- [ ] 注释是否与代码一致？（过时的注释比没有注释更糟糕）
- [ ] 组件 props 是否有类型和说明？
- [ ] 公共方法是否有 JSDoc 注释？

---

## 审查顺序建议

1. **快速通读**：先整体理解组件的功能和结构
2. **安全优先**：先扫描安全相关问题（类别1中的安全性）
3. **逻辑正确性**：检查核心逻辑和数据流
4. **性能隐患**：检查内存泄漏和渲染性能
5. **交互完整性**：检查用户反馈和边界处理
6. **视觉呈现**：检查样式和布局
7. **内容准确性**：检查文案和数据展示
